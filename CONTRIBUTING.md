# Contributing

1. Create a feature branch from `main`.
2. Never commit `.env`, credentials, customer data or provider secrets.
3. Keep Agents Morf provider-neutral and separate from product operational engines.
4. Do not modify the Grok Build source from this repository; integrate only through the restricted adapter contract.
5. Add or update tests for behavioral and API changes.
6. Run backend lint/tests and the frontend production build.
7. Open a pull request describing architecture, security, compatibility and migration impact.

Commit messages should be concise and imperative, for example: `Add feedback promotion workflow`.
